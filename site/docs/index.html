lantern documentation

Everything you need to build a game on lantern. The whole API is about 40 calls; you can read this page top to bottom in one sitting.

Install & build

# macOS (Linux: same packages via your package manager)
git clone https://github.com/alikatgh/lantern && cd lantern
brew install sdl2 lua cmake pkg-config
cmake -B build && cmake --build build -j8

./build/lantern games/showcase   # KORA NIGHT — complete mini-game
./build/lantern games/demo       # every feature in one scene
./build/lantern games/template   # the copy-me starter

Or grab the prebuilt bundle from the releases page — the engine binary with Lua statically linked and SDL bundled, no dependencies to install. See Distribution.

Your first game

A lantern game is a folder containing main.lua. Copy games/template/, rename it, and run it:

cp -r games/template games/mygame
./build/lantern games/mygame

Define two global functions. The engine calls them at 60 fps:

function update(dt)   -- dt = seconds since last frame (capped at 0.1)
end

function draw()       -- 3D first, then 2D composites on top
  lt.clear(0.1, 0.1, 0.2)
  lt.print("HELLO 400X240", 4, 4, 1, 1, 1, 1)
end

While the engine runs, edit and save main.lua — it reloads live. If you make a mistake, an in-engine error screen shows the file and line; fix it, save, and play resumes.

Core concepts

  • The screen is always 400×240 (lt.W × lt.H) — the 3DS top screen. It integer-upscales to the window with nearest filtering. There is no way to render at window resolution; that limit is the aesthetic.
  • 2D coordinates are pixels in that space, y-down. 3D is right-handed, y-up.
  • Draw order: 3D mesh and billboard calls render immediately, depth-tested. 2D calls (rect, sprite*, print) batch up and composite over the 3D at the end of the frame. Your HUD is always on top.
  • Lighting is per-vertex (gouraud) — the PICA200 idiom. Want round light pools on a floor? Tessellate it (lt.plane(12)). Want faceted low-poly shading? Keep meshes coarse.
  • Transparency in 3D is alpha-test: texels with alpha below 0.5 are discarded. No sorting, no halos, hard retro edges. (Blob shadows are the blessed exception — lt.shadow blends.)
  • The renderer is the engine's own software rasterizer — no GPU API. Texture sampling is bilinear by default (the real 3DS look); the built-in font and the final integer upscale stay nearest. Output is bit-identical across machines.
  • Handles are integers. Meshes, textures, and sounds return non-negative int handles; -1 (or a Lua error) means failure. Colors are floats 0..1 everywhere.

API — frame & screen

CallWhat it does
lt.W · lt.HScreen size constants: 400 and 240.
lt.clear(r,g,b)Clear color and depth. Call first in draw(). Match it to your fog color so distant geometry melts into the sky.
lt.time()Seconds since boot.
lt.screenshot(path)Save the current 400×240 frame as BMP.

API — 3D scene

CallWhat it does
lt.camera(ex,ey,ez, tx,ty,tz [,fov])Position the camera at (ex,ey,ez) looking at (tx,ty,tz). Default fov 55°. Settings persist between frames.
lt.light(dx,dy,dz [,ambient])The directional light (a sun) plus a flat ambient floor (default 0.35).
lt.point_light(i, x,y,z, radius [,r,g,b])Set point-light slot i (0–3 — four, like the 3DS hardware). Attenuation is (1−d/radius)², clamped. radius ≤ 0 turns the slot off.
lt.fog(start, end, r,g,b)Linear fog by view depth, applied to meshes and billboards. Disable with end ≤ start.
lt.draw(m, x,y,z, rx,ry,rz, sx,sy,sz [,r,g,b [,tex]])Draw mesh m with translation, rotation (radians), scale, an RGB tint, and an optional texture.
lt.billboard(t, x,y,z, w,h [,u0,v0,u1,v1])A camera-facing textured quad in the 3D scene — sprite characters in a diorama. Fullbright, fogged, alpha-tested. Optional UVs pick an atlas sub-rect.
lt.shadow(x,y,z, radius [,alpha])ALBW-style grounding shadow: a soft dark disc flat on the ground at height y (floor + small epsilon). Depth-tested, alpha-blended, never depth-written.
lt.draw_lerp(mA,mB,t, x,y,z, rx,ry,rz, sx,sy,sz [,r,g,b [,tex]])Keyframe animation, Quake-style: draws with vertices tweened between two same-count meshes, t = 0..1.

API — meshes

Vertex format: 12 floats — position (3), normal (3), UV (2), RGBA color (4). Vertex color multiplies texture and tint; it's your low-poly gradient tool.

CallWhat it does
lt.cube()Unit cube, per-face normals, 0..1 UVs per face.
lt.plane(segs)Unit XZ plane at y=0, tessellated segs×segs. More segments = rounder per-vertex light pools.
lt.sphere(seg) · lt.cylinder(seg) · lt.cone(seg)Unit-diameter primitives around the y axis; seg controls smoothness.
lt.mesh{…}Custom mesh from a flat table of vertices, 12 floats each, GL_TRIANGLES.
lt.load_mesh("model.obj")Wavefront OBJ (v/vt/vn/f, any face-index form, polygons fan-triangulate). Missing normals get flat face normals — the faceted look. Materials are ignored; texture via lt.draw.

API — 2D

CallWhat it does
lt.rect(x,y,w,h, r,g,b [,a])Filled rectangle with alpha blending.
lt.load_texture("img.bmp")Load a BMP, bilinear-filtered. Returns tex, w, h. Pure magenta (255,0,255) becomes transparent (classic color key), with automatic edge-bleed so filtering never fringes.
lt.sprite(t, x,y [,sx,sy])Draw a texture at its natural size × scale, top-left anchored.
lt.sprite_ex(t, cx,cy [,sx,sy, rot, r,g,b,a])Full-control sprite: center-anchored, rotation in radians, tint and alpha. Negative scale flips.
lt.sprite_uv(t, x,y,w,h, u0,v0,u1,v1)Draw an atlas sub-rect (UVs 0..1) into a destination rect — the tilemap workhorse.
lt.print("TEXT", x,y [,r,g,b,a])Built-in 8×8 uppercase font, 8 px advance, \n supported. Lowercase is upcased.

API — audio

A 48 kHz stereo mixer with 16 channels. WAVs convert once at load; loops are sample-exact.

CallWhat it does
lt.load_sound("bell.wav")Load a WAV (any common format/rate — converted at load).
lt.play(s [,volume, loop])Play on a free channel; returns the channel handle, or −1 if all 16 are busy. Pass true to loop (music).
lt.stop(ch)Stop a playing channel.
lt.volume(v)Master volume 0..1.

API — input

Keyboard and the first game controller are merged into one namespace: d-pad and left stick are the directions, A=z, B=x, Y=c, Start=return. Hotplug is handled.

CallWhat it does
lt.key(name)Held right now? Names: left right up down z x c space return escape a s d w.
lt.pressed(name)Went down this frame (edge-triggered — menus, jumps, relighting lamps).
lt.gamepad()Is a controller connected?
lt.quit()Request a clean exit (the loop ends this frame).
lt.escape_quits(false)By default the Escape key quits (dev convenience); disable it to run your own pause menu on lt.pressed("escape").
lt.rumble(low, high, ms)Rumble the pad (no-op without one). Intensities 0..1.
lt.touch()Touch position → x, y in 400×240 screen pixels (letterbox undone, clamped). Keeps the last touched position while nothing touches.
lt.touch_down()Is the screen touched right now? One point, 3DS-style — deliberately not multitouch. On desktop the left mouse button is the finger.
lt.touch_pressed()Touch began this frame. Latched, so even a tap shorter than one frame fires it exactly once.

API — saves & misc

CallWhat it does
lt.save(name, str)Persist a binary-safe string under name ([A-Za-z0-9_.-]). Stored in the OS application-support directory.
lt.load_save(name)Read it back, or nil if it doesn't exist.
-- hi-score in two lines
local best = tonumber(lt.load_save("best") or "0")
if score > best then lt.save("best", tostring(score)) end

The C ABI

Every Lua call above has a 1:1 twin in include/lantern.h with an lt_ prefix — the Lua host is itself just a client of that header. Link the lantern_core static library and you have the whole engine from C or C++:

#include <lantern.h>

static void update(double dt) { }
static void draw(void) {
    lt_clear(0.1f, 0.1f, 0.2f);
    lt_print("HELLO FROM C", 4, 4, 1, 1, 1, 1);
}

int main(void) {
    if (!lt_boot("my game", 3)) return 1;
    lt_run(update, draw);         /* or drive the loop yourself:      */
    lt_shutdown();                /* lt_frame_poll/_begin/_end/_dt    */
}

games/hello_c/main.c is a complete working example, built as the lantern_hello_c CMake target.

Assets

  • Textures — BMP. Nearest-filtered, clamped. Pure magenta (255,0,255) reads as transparent. Keep them small; 256×256 is the spiritual maximum.
  • Sounds — WAV. Any common encoding; converted to 48 kHz float at load.
  • Models — OBJ. Omit normals on purpose if you want flat faceted shading — the loader computes face normals.
  • Asset paths in Lua are relative to the game folder.

The dev loop

  • Hot-reload: the engine watches main.lua (checked twice a second). Save the file and it re-runs in the same Lua state — globals persist, resources re-create.
  • Error screen: a Lua error (load-time or runtime) renders in-engine with the file, line, and message. The watcher keeps running: fix the file, save, and the game resumes.
  • Uncapped mode: LANTERN_NOVSYNC=1 disables vsync for benchmarking (see games/stress/ — ~10,300 lit draw calls fit the 16.6 ms budget on an M-series laptop).
  • Deterministic mode: LANTERN_FIXED_DT=1 makes dt and lt.time() advance by a fixed step — repeated screenshot runs become byte-identical, on any machine. Use it for CI captures.
  • Hot-reload is leak-free: every reload frees the previous generation's meshes, textures, and sounds before re-running main.lua.

Verification (screenshot your game in CI)

LANTERN_SHOT=/tmp/myshot ./build/lantern games/mygame
# runs 60 frames, writes /tmp/myshot.bmp (the real 400×240 frame), exits 0

LANTERN_SHOT_FRAME=300 ...   # capture a later frame instead

This is how lantern itself is developed: every engine feature lands with a captured frame. Combine it with a deterministic auto-play mode in your game (see games/showcase/'s SHOWCASE_AUTO) and you can screenshot-test full gameplay in CI.

Distribution

bash tools/make_dist.sh
# → dist/lantern-<ver>-macos-<arch>.tar.gz  (~1.3 MB)

The bundle contains the engine binary (Lua statically linked, SDL bundled and rewired to @executable_path), all sample games, lantern.h, and the docs. Recipients need no package manager and no compiler. The script refuses to package itself unless the bundled binary actually renders a frame.
